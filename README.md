# xv6-RISCV-Enhanced
本项目以基于 RISC-V 指令集架构的 xv6 操作系统为基础，深入分析了系统调用、内存管理、进程管理、中断、文件系统的实现 过程，并对其进行了扩展和优化。

1. 实现 Copy-On-Write Fork 机制。利用页表权限位触发中断，结合物理页生命周期引用计数，实现 Fork 系统调用的懒拷贝机制。

2. 设计并实现了 mmap 和 munmap 系统调用。引入了虚拟内存区域 VMA 追踪映射状态，并支持了脏页的按块回写，实现了文件的按需加载与文件到内存的直接映射。

3. 设计了基于伙伴系统与 slab 分配器的内存碎片控制方案。引入了伙伴系统，支持物理内存块的动态分裂与合并，解决了外部碎片问题；并且引入了 slab 分配器，支持了内核常用小对象（如进程描述符、文件表项）的细粒度分配，有效优化了内部碎片问题。

4. 内存高负载条件下的锁争用优化。为内存管理设计了 Per-CPU 缓存架构，引入 PCP 批量填充与回收机制大幅降低全局锁请求次数，引入跨核工作窃取算法平衡内存水位。在内存耗尽场景中将全局锁冲突从74万次降低到130次左右，同时也进一步提升了CPU缓存亲和性。

5. 高并发场景下的写者饥饿与锁争用优化。设计并实现了支持写者优先的高效读写锁，采用基于 32 位整型与位掩码的状态压缩机制，最大程度降低了缓存失效带来的性能损失，结合原子变量与 CAS 原语实现了读写锁内部无锁化。

6. E1000 网卡底层驱动开发。通过配置 DMA 的发送 TX 和接受 RX 环形队列，结合网卡硬件中断机制，完成了底层网络数据包的高效收发机制，为上层的网络协议栈提供了底层支持。进一步为系统引入了 ARP、UDP、TCP 、HTTP 协议的部分支持。
